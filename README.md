# IngredientConservo

https://nathaferrari.github.io/IngredientConservo-Deploy

Your pantry, your legacy. A Flutter app to track groceries, reduce food waste, and generate recipes from what you already have.



--------------------------------------------------------

1. coupon_catalog
A catalog of available discount coupons/rewards that users can redeem with points.

region (varchar, required, indexed) — Geographic region (e.g., "us").

section (varchar, required) — Category section (e.g., "eco", "restaurants").

store (varchar, required) — Store/brand name (e.g., "Too Good To Go", "Chipotle").

emoji (varchar, required) — Visual icon for the coupon.

discount (varchar, required) — Discount description/offer.

description (text, required) — Longer description of the offer.

pointsCost (integer, required, 0–10000) — Loyalty points needed to redeem.

expiryDays (integer, 1–365, default: 30) — How many days the coupon is valid after redemption.

colorHex (varchar, default: #3F5E...) — UI color for the coupon card.

active (boolean, indexed, default: true) — Whether the coupon is currently available.

sortOrder (integer, indexed, default: 0) — Display order in the UI.

$id, $createdAt, $updatedAt — Auto-generated metadata.

2. pantry_events
An event log tracking pantry-related activities (e.g., items added, consumed, expired).

ownerId (varchar, required, indexed) — Links to the user.

timestamp (datetime, required, indexed) — When the event occurred.

name (varchar, required) — Name/description of the event.

category (varchar, required) — Event category.

kind (varchar, required) — Type of event (e.g., "add", "consume", "expire").

daysLeftAtEvent (integer, required) — Days remaining until expiration at event time.

$id, $createdAt, $updatedAt — Auto-generated metadata.

3. recipe_history
Stores recipes generated by the AI for users, serving as a history of suggested meals.

ownerId (varchar, required, indexed) — Links to the user.

title (varchar, required) — Recipe name.

description (text, required) — Brief recipe description.

culture (varchar, required) — Cuisine type (e.g., "Swiss", "Italian").

ingredientsUsed[] (varchar array, default: NULL) — Ingredients from pantry used in the recipe.

priorityIngredientsUsed[] (varchar array, default: NULL) — Ingredients prioritized for use (e.g., expiring soon).

missingIngredients[] (varchar array, default: NULL) — Ingredients needed but not in pantry.

steps[] (text array, default: NULL) — Cooking instructions as a list.

generatedAt (datetime, required, indexed) — When the recipe was generated.

$id, $createdAt, $updatedAt — Auto-generated metadata.

4. recipe_favorites
Recipes that users have saved/liked for quick access later.

ownerId (varchar, required, indexed) — Links to the user.

title (varchar, required) — Recipe name.

description (text, required) — Brief recipe description.

culture (varchar, required) — Cuisine type.

ingredientsUsed[] (varchar array, default: NULL)

priorityIngredientsUsed[] (varchar array, default: NULL)

missingIngredients[] (varchar array, default: NULL)

steps[] (text array, default: NULL)

savedAt (datetime, required, indexed) — When the user favorited the recipe.

$id, $createdAt, $updatedAt — Auto-generated metadata.

5. rewards
Discount/reward coupons that users have already redeemed.

ownerId (text, required) — Links to the user.

couponCode (varchar, required) — Unique coupon code.

storeName (varchar, required) — Store name (e.g., Migros, Alnatura, Too Good To Go).

discount (varchar, required) — Discount value.

pointsCost (integer, required) — Points spent to redeem.

redeemedAt (datetime, required) — When redeemed.

expiresAt (datetime, required) — Coupon expiration date.

$id, $createdAt, $updatedAt — Auto-generated metadata.

6. user_profile
User-specific data for gamification, loyalty, and personalization.

ownerId (text, required) — Appwrite user ID.

country (varchar, required) — User's country.

points (integer, default: 0) — Current loyalty points balance.

totalSpent (double, default: 0) — Total money tracked.

totalWasted (double, default: 0) — Food waste avoided.

appLanguage (varchar, default: NULL) — User's preferred language (NEW).

mascotTourCompleted (boolean, default: false) — Whether the onboarding tour was completed (NEW).

mascotTourStepIndex (integer, default: NULL, 0–100) — Current step in the onboarding tour (NEW).

$id, $createdAt, $updatedAt — Auto-generated metadata.

7. item
Food/grocery items in users' pantries/inventories.

name (text, required) — Item name.

emoji (text, required) — Icon representing the item.

expirationDate (datetime, required) — When the item expires.

category (text, required) — Food category.

ownerId (varchar, required) — Links to the user.

price (double) — Price of the item.

unit (varchar, required) — Currency or unit (e.g., CHF).

quantity (double, default: 0) — Quantity in stock.

measure (text, default: "u") — Measurement unit.

$id, $createdAt, $updatedAt — Auto-generated metadata.

Functions
1. ReceiptHandler
Purpose: Processes receipts (scans/parses receipt images) and automatically creates item entries from them. Uses Groq AI APIs.

Runtime: Node.js 25, Entrypoint: src/main.js

Git repo: receipt-handler (main branch), auto-deploys from GitHub.

Env vars: GROQ_API_KEY, HF_TOKEN

Schedule: Runs every minute (* * * * * cron).

Timeout: 15s

Access: Any user

Resources: 0.5 CPU, 512 MB RAM

2. recipe_generator
Purpose: Generates recipe suggestions based on available ingredients. Uses Hugging Face AI APIs.

Runtime: Node.js 25, Entrypoint: src/main.js

Git repo: recipes-creator (main branch), auto-deploys from GitHub.

Env vars: OCR_API_KEY, HF_TOKEN

Schedule: None (triggered on-demand).

Timeout: 15s

Access: Any user

Resources: 0.5 CPU, 512 MB RAM
